# Project Summary
The project is an F1 analytics application designed to provide users with in-depth insights into Formula 1 racing data. It features robust user account management, including login, signup, profile management, and settings. The application enhances user experience by offering detailed analytics and a user-friendly dashboard to navigate race statistics and standings.

# Project Module Description
- **User Account Management**: Includes login, signup, profile management, settings, and password recovery.
- **Dashboard**: Displays race statistics, standings, and relevant analytics.
- **Routing**: Manages navigation across different application views.
- **Authentication**: Implements user authentication using Supabase, including login, signup, password reset, and session management.
- **Protected Routes**: Secures specific routes to ensure access is limited to authenticated users.

# File Description Inventory
- **docs/**: Contains documentation files, including design and architecture diagrams.
- **src/components/**: Reusable UI components such as buttons, cards, forms, and layout components.
- **src/contexts/**: Contains `AuthContext.tsx` for managing authentication state.
- **src/pages/**: Application pages including LoginPage, SignupPage, ProfilePage, SettingsPage, ForgotPasswordPage, and PasswordResetPage.
- **src/App.tsx**: Main component setting up application routing and context providers.
- **src/components/layout/Header.tsx**: Header component providing navigation and user menu functionality.

# Technology Stack
- **React**: Frontend framework for building user interfaces.
- **TypeScript**: Superset of JavaScript that provides type safety.
- **Vite**: Build tool for faster development.
- **Tailwind CSS**: Utility-first CSS framework for styling.
- **React Router**: For managing routing and navigation.
- **Tanstack Query**: For data fetching and state management.
- **Supabase**: For authentication and database management.

# Usage
1. **Install Dependencies**: Run `pnpm install` to install all necessary packages.
2. **Build the Application**: Execute `pnpm run build` to create a production build.
3. **Run the Application**: Use `pnpm run start` to start the development server.
