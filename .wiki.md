# Project Summary
The F1 Fan Analytics Platform is a dynamic application designed for Formula 1 enthusiasts, providing real-time insights, analytics, and predictive capabilities regarding races, drivers, and constructors. It aims to enhance user engagement through an intuitive interface and personalized experiences, leveraging advanced machine learning and data from the FastF1 API. This platform is essential for fans and analysts seeking in-depth race data and predictive analytics, making it a vital tool for both casual viewers and serious analysts in the F1 community.

# Project Module Description
- **User Account Management**: Manages user authentication, profile settings, and preferences.
- **Dashboard**: Presents live race statistics, standings, and analytics without loading states.
- **AI Assistant**: Offers intelligent insights and support powered by OpenAI.
- **Race Countdown**: Displays the time remaining until the next race.
- **Race Schedule**: Lists upcoming races with their respective dates and times.
- **Driver and Constructor Standings**: Shows current rankings for drivers and constructors.
- **Last Race Results**: Provides results from the most recent race.
- **Strategy Simulator**: Allows users to simulate race strategies and optimize tire choices.
- **Predictions Tab**: Displays machine learning-generated predictions for upcoming races.
- **Admin Panel**: Facilitates backend management and automatic data synchronization.

# File Description Inventory
- **docs/**: Contains comprehensive documentation files, including technical specifications and implementation roadmaps.
- **src/components/**: Reusable UI components for dashboards, user interfaces, and loading states.
- **src/contexts/**: Contains context management files for user authentication and state management.
- **src/pages/**: Application pages including Index, AdminPage, LoginPage, SignupPage, ProfilePage, and SettingsPage.
- **src/App.tsx**: Main component responsible for routing and context providers.
- **src/lib/**: Utility files for data fetching and backend operations.

# Usage
1. **Install Dependencies**: Run `pnpm install` to install all necessary packages.
2. **Initialize Backend Data**: The application automatically syncs data upon startup. Ensure the FastF1 library is installed for data fetching.
3. **Start Development Server**: Use `pnpm run dev` to start the application.
4. **Build for Production**: Execute `pnpm run build` to create a production build.
